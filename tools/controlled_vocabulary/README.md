# Controlled Vocabulary Template

A template and tooling for defining and enforcing a controlled vocabulary
within a codebase, as described in Section V.

## Purpose

Provides the semantic pillar intervention. A short, maintained,
authoritative list of primitive terms used in a codebase, each defined
with enough precision to distinguish it from every term it might be
confused with.

## Contents

- Template file structure for vocabulary definitions.
- Example vocabulary for a data ingestion domain (load, fetch, parse,
  read, ingest, download, deserialise).
- Validation script to check codebase naming against the vocabulary.
